How it works

Three steps. No YAML required.

01 · DROP
Drop a folder

Point DROP at any project directory. No Dockerfile, no build script, no config file needed to begin.

02 · DETECT
Auto-detect & build

DROP recognizes the runtime and framework — Next.js, FastAPI, Express and more — then installs and builds automatically.

03 · DEPLOY
Get a URL

Your app comes online at myapp.localhost with a persistent port, PM2 process management, and automatic HTTPS.

Escape hatch

Zero config by default. Full control when you want it.

80% of apps need nothing. For the other 20%, drop a drop.yaml to pin runtime, custom domains, env, and persistent data directories that survive upgrades.

▸Pin runtime & framework
▸Custom per-app domains
▸Inject environment variables
▸Persistent data dirs (DROP_DATA_DIR)
drop.yaml
name: myapp
runtime: node
domains:
  - app.example.com
database: postgres
env:
  NODE_ENV: production
persist:
  - ./uploads
  - ./data
